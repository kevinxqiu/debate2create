# Documentation

The top-level `README.md` covers installation, smoke tests, configuration, and
main entry points. These additional notes document optional subsystems:

- `PLURALISTIC_JUDGE_README.md`: panel-based LLM judge feedback used by the
  Debate2Create loop.
- `PLOTTING_README.md`: plotting utilities for completed debate runs.

Internal development summaries, paper builds, and experiment outputs are not
kept in this directory.

Third-party code and asset attribution is listed in
`../THIRD_PARTY_NOTICES.md`.
